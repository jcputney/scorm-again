SCORM Again - Interactive Demos

1. Interactive Demo

A comprehensive demo showing basic SCORM functionality including API usage, CMI data model interaction, progress tracking, and score reporting.

  • Basic SCORM API usage
  • CMI data model interaction
  • Progress tracking
  • Score reporting
  • Completion status

2. Selection and Randomization Demo

Demonstrates SCORM 2004 advanced sequencing features for selecting and randomizing activities in a course structure.

  • Select Count (SR.1) - Limiting number of child activities
  • Randomize Children (SR.2) - Randomizing order of activities
  • Different timing options (once, on each new attempt)
  • Visual representation of activity tree changes
Launch Demo

3. Exit Action Rules Demo

Demonstrates SCORM 2004 exit action rules (TB.2.1) that control how activities are terminated based on specific conditions.

  • Exit - Terminates current activity
  • Exit Parent - Terminates parent activity and all its children
  • Exit All - Terminates all activities in the course
  • Conditional exit rules based on activity state
  • Visual feedback showing activity termination
Launch Demo

4. Post-Condition Rules Demo

Demonstrates SCORM 2004 post-condition rules (TB.2.2) that automatically trigger navigation actions after an activity completes.

  • Continue - Automatically move to next activity
  • Previous - Go back to previous activity
  • Retry - Retry the current activity
  • Retry All - Start the course over
  • Exit Parent/All - Exit based on completion
  • Conditional rules based on completion, satisfaction, attempts, and scores
